# Step 4: Commit and Push Changes

## 1. Make Changes

Edit, add, or delete files in your local repo folder as normal.

## 2. Stage Changes

```bash
git add file.txt        # stage one file
git add .                # stage everything
```

## 3. Commit

```bash
git commit -m "Describe what you changed"
```

A commit is a checkpoint — write clear, short messages explaining *why*, not just *what*.

## 4. Push to GitHub

```bash
git push origin main
```

This uploads your commits to the remote repository on GitHub.

## 5. Pull Latest Changes

Before pushing, it's good practice to pull any updates others may have made:

```bash
git pull origin main
```
