# Step 5: Branches and Pull Requests

Branches let you work on changes without affecting the main codebase.

## 1. Create a Branch

```bash
git checkout -b feature/my-update
```

## 2. Make and Push Changes

```bash
git add .
git commit -m "Add my update"
git push origin feature/my-update
```

## 3. Open a Pull Request (PR)

1. Go to your repo on GitHub — you'll see a prompt to **Compare & pull request**.
2. Click it, add a title and description explaining the change.
3. Select the **base branch** (usually `main`) and **compare branch** (your feature branch).
4. Click **Create pull request**.

## 4. Review and Merge

- Reviewers can comment, request changes, or approve.
- Once approved, click **Merge pull request**.
- Delete the feature branch afterward (GitHub offers a button for this) to keep things tidy.

## 5. Sync Your Local Main

```bash
git checkout main
git pull origin main
```
