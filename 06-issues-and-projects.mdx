# Step 6: Issues and Project Boards

## Issues

Issues track bugs, tasks, and feature requests.

1. On your repo, go to the **Issues** tab → **New issue**.
2. Add a title and description (steps to reproduce, expected behavior, etc.).
3. Optionally assign **labels**, an **assignee**, and a **milestone**.
4. Click **Submit new issue**.

Link a commit or PR to an issue by referencing it:

```bash
git commit -m "Fix login bug, closes #12"
```

This automatically closes issue #12 when the commit is merged into the default branch.

## Project Boards

Project boards organize issues and PRs visually (e.g. Kanban-style: To Do / In Progress / Done).

1. Go to the **Projects** tab → **New project**.
2. Choose a template (Board, Table, or Roadmap).
3. Add issues/PRs as cards, and drag them between columns as work progresses.
